4. Development Approach & Steps
Phased build plan — from foundations to full deployment
Phase 1 · Foundation (Week 1–2)

🏗️ Infrastructure & Data Layer

  • Set up Google Sheets Master Schema (all required + optional fields, status flags)
  • Create Google Drive folder hierarchy template (auto-provision per event)
  • Configure credential vault: n8n / Make.com / .env with OAuth tokens
  • Implement Google Sheets MCP connector (read/write, row-level locking)
  • Build IST-aware timestamp utilities and field validation library
  • Write missing-field detection + email alert template (Appendix B)
Google Sheets API
Google Drive API
Python / pytz
SMTP
Phase 2 · Orchestrator (Week 2–3)

🎯 Orchestrator Agent Core

  • Build state machine: 12 stages with transition guards and idempotency via status flags
  • Implement row-change trigger (polling or webhook on Sheet)
  • Design agent dispatch protocol: async task queue with retry logic
  • Write-back subsystem: update flags, URLs, timestamps atomically
  • Error handling: catch, log, notify organizer; never leave partial state
  • Unit test all transitions including missing-field and retry paths
LangGraph / CrewAI
Python asyncio
Redis (state)
pytest
Phase 3 · Event Creation (Week 3–4)

📅 Meetup + QR + Poster Agents

  • Meetup Agent: OAuth 2.0 flow, create/publish event, return URL+ID, headless fallback
  • QR Agent: qrcode library, generate PNG, upload to Drive, return URL
  • Poster Agent: brand-safe layout engine (Pillow/Canva API), all zones per spec
  • Implement approval loop: send Drive link → poll for response → rework or proceed
  • Meetup poster upload after approval (closes the cyclic dependency loop)
  • Naming convention enforcement (Appendix D)
Meetup OAuth API
qrcode lib
Pillow / Canva API
Make.com
Phase 4 · Social & Comms (Week 4–5)

📢 Social Syndication + WhatsApp + Partner Agents

  • Social Agent: integrate LinkedIn, Facebook, X, Instagram APIs via Make.com/Buffer
  • Message templates with variable substitution (Appendix A templates A1–A4)
  • Save all drafts to /03_Social_Copies/ before posting
  • WA Helper: Selenium/WhatsApp Web semi-automation, confirm-per-group flow
  • Partner Agent: personalized email via SendGrid, UTM parameter generation
  • Sheet flags: Announce_Sent?, Whatsapp_Groups_Posted?, Partners_Notified?
Buffer / Hootsuite
LinkedIn API
Selenium
SendGrid
Phase 5 · Conferencing & Reminders (Week 5–6)

🎥 Conferencing + ⏰ Reminders Agents

  • Conferencing Agent: Zoom/Teams API or GMeet (sponsor licensed), apply security checklist
  • Conditional: if Online/Hybrid → inject link into Meetup description
  • Reminders Agent: cron jobs in IST for T-2d / T-1d / T-2h
  • Each reminder dispatches Social + WA + Meetup Contact Attendees in parallel
  • T-2h adds conference link if mode is Online or Hybrid
  • Guard all sends with flag checks to prevent duplicates (idempotent)
Zoom API
MS Teams API
APScheduler
Celery
Phase 6 · Post-Event (Week 6–7)

✍️ Post-Event + Event Execution Agents

  • Event Exec Agent: Google Forms for speaker material upload pre-event
  • Verify-on-start checklist: recording ✅ transcript ✅ chat ✅
  • Post-Event Agent: transcript → LLM (Claude/GPT) → structured blog output
  • LinkedIn post builder: photo selection, gratitude copy, C-level tagging
  • Enforce closed-sharing rule: Recording/Transcript/Slides → Meetup + closed WA only
  • Meetup photo album upload + consolidated resources post
  • Finalize all sheet links and set PostEvent_Completed? = Y
Claude / GPT API
Google Forms
Drive API

Phase 7 · Analytics (Week 7–8)

📊 Analytics Agent + Dashboard

  • Aggregate per-event KPIs from Sheet + platform APIs into analytics table
  • Build Looker Studio / Metabase dashboard: funnel, engagement, ops cadence, segmentation
  • Automated completion email with KPI snapshot to organizer
  • Month-over-month growth tracking toward 10,000 Meetup members goal (March 2026)
  • Partner uplift measurement via UTM data
Looker Studio
Metabase
BigQuery
pandas
Phase 8 · Hardening (Week 8–9)

🔐 Security, Testing & Deployment

  • Credential rotation policy (~60 days); 2FA on primary organizer accounts
  • Closed-sharing enforcement tests: assert no Recording/Transcript/Slides on public social
  • End-to-end integration tests for full event lifecycle (dry-run mode)
  • Error simulation: missing fields, API failures, approval rejections, retry exhaustion
  • Rate-limit handling for all APIs (Meetup free tier, social platforms)
  • Deploy to cloud (GCP / Railway) with monitoring + alerting
